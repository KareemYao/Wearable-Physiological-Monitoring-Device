## 💡 项目简介

本项目旨在构建一套**小型化、低成本**的可穿戴健康监测方案：

1. 通过 **MAX30102** 传感器采集红光（Red）与红外光（IR）PPG 信号，实时计算**心率（HR）**与**血氧饱和度（SpO₂）**。
2. 通过 **NTC 热敏电阻** + ADC 采集环境/体温。
3. 通过 **电阻分压** + ADC 监测锂电池电压，结合温度信息进行**二维查表**精准估算电池剩余百分比。
4. 所有数据经 **UART DMA** 以固定帧格式周期性上报至上位机或蓝牙模块。

## 📊 核心算法亮点

### 1. MAX30102 心率血氧算法 (`bsp_ir_red_cal2.c`)

> 在仅 **8KB RAM** 的严苛约束下，未使用传统的 FFT 或自相关等高资源消耗算法，而是独创了一套**轻量级时域分析方案**。

#### 🔹 Union 联合体内存复用

```c
typedef union {
    struct { uint32_t raw_red[200]; uint32_t raw_ir[200]; ... } stage_filter;   /* 阶段A: 采集滤波 */
    struct { uint8_t pos_idx[200];  float pos_val[200];  ... } stage_peaks;    /* 阶段B: 寻峰寻谷 */
} ReusableMem_t;
```

采集/滤波阶段与寻峰阶段的生命周期**严格错开**，通过 `union` 将两组大数组映射到**同一块物理内存**，节省近 **50%** 的 RAM 占用。

#### 🔹 局部极值寻找法（正峰与负谷融合）

- 将 IR 交流分量分为**正半轴（Positive Peaks）**与**负半轴（Negative Valleys）**，分别独立寻找两个最显著的极值点。
- 通过极值点的时间间隔计算心率：$HR = \frac{60 \times f_s}{\Delta n}$，其中 $f_s = 100\text{Hz}$。
- 正峰心率与负谷心率**交叉验证**，仅当两者吻合时采信，单侧异常时自动降级为单源，双侧均离谱则**拒收脏数据**。

#### 🔹 完美环形队列校准机制

```
首次校准 → 等待正峰/负谷心率同时落入 [60, 90] 的"完美静息域"
         → 以此作为基准锚点写入环形缓冲区第 0 位
后续更新 → 新数据与上一次输出偏差 ≤ 10bpm 才准入
         → 环满后取平均值回写第 0 位，形成新基准
         → 偏差过大的数据直接丢弃，维持上次稳态值
```

- 配合 **3 点滑动平均滤波**（`HR_SlidingAvgFilter` / `SPO2_SlidingAvgFilter`），实现了极高的**抗运动伪影能力**，杜绝心率突变与跳零现象。

#### 🔹 SpO₂ 计算

基于经典的比值-比值法（Ratio of Ratios）：

$$R = \frac{AC_{Red} / DC_{Red}}{AC_{IR} / DC_{IR}}, \quad SpO_2 = 104 - 17 \times R$$

### 2. 电池监测算法 (`bsp_adc_dma.c`)

#### 🔹 NTC 查表法测温

- 预置 **-10°C ~ 50°C**（每 10°C 一个节点）的 NTC 分压电压查找表。
- 根据当前 ADC 采样电压进行**线性插值最近邻匹配**，快速求解温度。

#### 🔹 温度-电压二维 LUT 电量估算

```
        4.2V   4.0V   3.8V   3.6V   3.5V   3.3V
-10°C [  90%    50%    15%     0%     0%     0% ]
  0°C [  95%    70%    40%    10%     0%     0% ]
 10°C [ 100%    80%    55%    20%     5%     0% ]
 20°C [ 100%    85%    60%    25%    10%     0% ]
 30°C [ 100%    90%    65%    30%    10%     0% ]
   …
```

- 综合考虑**低温容量衰减**特性，在不同温度下对同一电压给出差异化的电量百分比，避免冬季场景下的误报。

#### 🔹 LED 电量指示策略

| 电量范围 | 绿灯 (PC6) | 红灯 (PC7) | 说明 |
|:--------:|:----------:|:----------:|:-----|
| > 50% | ✅ 亮 | ❌ 灭 | 正常使用 |
| 20%~50% | ❌ 灭 | ❌ 灭 | 省电模式 |
| < 20% | ❌ 灭 | ✅ 亮 | 低电预警 |

## ⚙️ 软件架构与外设配置

### 系统运行流程

```
main()
  ├── HAL_Init() / SystemClock_Config()     ← HSI 16MHz → PLL → 64MHz
  ├── MX_GPIO_Init()
  ├── MX_DMA_Init()
  ├── MX_ADC1_Init()
  ├── MX_TIM1_Init()
  ├── MX_USART1_UART_Init()                ← 115200, 8N1
  ├── HR_Device_Init()                      ← 软 I2C 配置 MAX30102 寄存器
  ├── BSP_TIM_Init(&htim1) + Start_IT()    ← 启动 10ms 定时器
  └── while(1)
        └── if (adc_data_ready)
              └── Task_Battery_Monitor()    ← 读 ADC → 测温 → 查表 → LED 控制
```

### 外设配置明细

| 外设 | 角色 | 关键参数 |
|:-----|:-----|:---------|
| **TIM1** | 10ms 周期中断 | 驱动软 I2C 读取 MAX30102 FIFO（每次 6 字节） |
| **ADC1 + DMA** | 多通道循环采集 | CH1(PA1) = NTC，CH2(PA2) = 电池电压；DMA 交错存储，20 次采样取均值 |
| **USART1 + DMA TX** | 异步非阻塞发送 | 115200 波特率；发送完成中断回调释放总线 |
| **Soft I2C** | MAX30102 通信 | PB3(SCL) / PB6(SDA)，开漏输出 + 内部上拉；设备地址 0x57 |

### 定时器中断任务调度

```
TIM1 IRQ (每 10ms)
  ├── 读取 MAX30102 FIFO (6 Bytes)
  ├── MAX30102_ProcessData()
  │     ├── 累积至 200 点 → 批量滤波 + AC/DC 分离
  │     ├── 寻峰寻谷 → 计算 HR / SpO₂
  │     └── 环形校准 + 滑动平均
  └── 每 2 秒 (200 × 10ms)
        ├── Send_DataPacket()       ← UART DMA 发送
        └── BSP_ADC_StartDMA()      ← 触发一次 ADC 批量采集
```

## 📡 通信协议

### 数据帧格式（6 字节定长）

| 字节偏移 | 字段 | 值 | 说明 |
|:--------:|:-----|:--:|:-----|
| 0 | Header 1 | `0xAA` | 帧头 |
| 1 | Header 2 | `0x55` | 帧头 |
| 2 | HR | 0~200 | 心率值 (bpm) |
| 3 | SpO₂ | 90~99 | 血氧饱和度 (%) |
| 4 | Checksum | — | `SUM(Byte[0..3]) & 0xFF` |
| 5 | Tail | `0xBB` | 帧尾 |

**发送周期**：每 **2 秒** 发送一帧。

**示例帧**：

```
AA 55 48 62 A9 BB
       │  │  └─ Checksum: 0xAA+0x55+0x48+0x62 = 0x1A9 → 0xA9
       │  └──── SpO₂ = 98%
       └─────── HR = 72 bpm
```
